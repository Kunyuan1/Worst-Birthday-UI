# Worst Birthday UI

An intentionally frustrating birthday input interface designed to challenge users through poor usability and unconventional interaction patterns.

This project is inspired by the idea of “bad UI battles,” where developers create deliberately inefficient or confusing interfaces to highlight what not to do in user experience design.

## Features

* No standard date picker
* Must win blackjack for a month
* Figure out a combination of numerical equations for the date
* Requires precision, timing, or trial-and-error to select a year
* Intentionally confusing and inefficient design
* Built for demonstration and entertainment purposes

## How to Run Locally

1. Clone the repository:

```bash
git clone https://github.com/Kunyuan1/Worst-Birthday-UI.git
cd Worst-Birthday-UI
```

2. Open the project:

* Open `index.html` directly in your browser
  or
* Use a development server such as VS Code Live Server

## Purpose

This project was created to:

* Demonstrate poor UI/UX design practices
* Highlight the importance of usability in interface design
* Explore creative front-end interactions
* Provide an engaging and humorous user experience

---

## Disclaimer

This interface is intentionally designed to be difficult to use.

It should not be used as a reference for real-world applications.

---

## Lessons Learned

* Simplicity improves usability
* Predictability is essential for good user experience
* Users should have clear control over interactions
* Poor design leads to frustration and disengagement
